# Reddit 참고 레이아웃 구현 계획

> 목적: Reddit을 그대로 복제하지 않고, 레이아웃 흐름만 참고해서 ALIBAI 게시판의 메인/상세 화면을 확장한다.
> 디자인 단계가 아니라 기능 배치와 데이터 흐름을 정리하는 구현 계획이다.

---

## 1. 목표

### 메인 화면

가져올 것:

- 상단 검색창
- 가입 버튼
- 로그인 버튼
- 게시글이 세로로 이어지는 feed 방식

가져오지 않을 것:

- Reddit 로고/색/세부 디자인 복제
- 왼쪽 가입 유도 패널
- 오른쪽 인기 커뮤니티 패널
- 광고/추천 커뮤니티/푸터 링크

### post 상세 화면

가져올 것:

- 댓글 작성창
- 댓글 목록
- 댓글 좋아요
- 게시글 투표수
- 공유 버튼

## 2. 현재 상태와 구현 기준

### 이미 있는 백엔드 계약

- `GET /posts?q=&tag=&cursor=`
  - 게시글 목록 조회
  - 검색어는 `q`로 전달
- `GET /posts/{id}`
  - 게시글 상세 조회
  - 응답에 `score`, `myVote` 포함
- `POST /posts/{id}/vote`
  - 요청 body: `{ "value": 1 | -1 }`
  - 응답: `{ "score": number, "myVote": 1 | -1 | 0 }`
  - 같은 값 재클릭은 취소, 반대 값 클릭은 전환
- `GET /posts/{id}/comments`
  - 댓글 목록 조회
- `POST /posts/{id}/comments`
  - 댓글 작성
  - 로그인 필요
- `POST /auth/signup`
  - 가입
- `POST /auth/login`
  - 로그인

### 이번 구현에서 추가할 백엔드 계약

댓글 좋아요를 실제 기능으로 만들기 위해 기존 `votes` 테이블을 재사용한다.

추가 엔드포인트:

```text
POST /comments/{comment_id}/like
```

동작:

- 로그인 필요
- 아직 좋아요를 누르지 않았으면 좋아요 추가
- 이미 좋아요를 눌렀으면 좋아요 취소
- 응답은 아래 모양으로 통일

```json
{
  "likeCount": 3,
  "myLike": true
}
```

댓글 목록 응답도 아래 필드를 포함하도록 확장한다.

```json
{
  "id": 1,
  "body": "댓글 내용",
  "authorId": 1,
  "createdAt": "2026-06-13T12:00:00Z",
  "likeCount": 3,
  "myLike": false
}
```

## 3. 프론트 구현 계획

### 3.1 메인 레이아웃

`PostListPage`를 Reddit feed 느낌의 중앙 리스트 화면으로 정리한다.

구조:

```text
상단 헤더
  왼쪽: 서비스 이름
  가운데: 검색창
  오른쪽: 가입 / 로그인

메인 영역
  중앙 feed
    post card
    post card
    post card
```

로그인 상태별 버튼:

- 비로그인: `가입`, `로그인`
- 로그인: `새 글`, `로그아웃`

검색 흐름:

1. 검색 input 값을 state로 관리한다.
2. Enter 또는 검색 버튼 클릭 시 `fetchPosts({ q })`를 호출한다.
3. 응답의 `items`를 `posts` state에 넣는다.
4. 검색 중에는 로딩 상태를 보여준다.

### 3.2 post card

`PostCard`는 feed 안의 한 게시글 단위로 정리한다.

표시 요소:

- 제목
- 본문 일부 또는 전체
- 태그 목록
- 투표수
- 댓글/공유 자리

주의:

- card 전체 디자인은 단순하게 유지한다.
- Reddit처럼 이미지 중심 레이아웃을 만들지 않는다.
- 현재 ALIBAI 데이터에는 이미지 필드가 없으므로 이미지 영역을 만들지 않는다.

### 3.3 post 상세 레이아웃

`PostDetailPage`는 아래 순서로 정리한다.

```text
back 버튼
게시글 본문 영역
  제목
  본문
  태그

게시글 액션 영역
  up vote 버튼
  score
  down vote 버튼
  공유 버튼

댓글 작성 영역
댓글 목록
  댓글 1
    내용
    작성 시각
    좋아요 버튼
    좋아요 수
  댓글 2
```

게시글 투표 흐름:

1. `Post` 타입에 `score`, `myVote`를 추가한다.
2. `api/posts.ts`의 `toPost`가 백엔드 `score`, `myVote`를 보존하게 한다.
3. `votePost(postId, value)` 함수를 추가한다.
4. 상세 페이지에서 투표 클릭 시 `votePost`를 호출한다.
5. 응답의 `score`, `myVote`로 현재 `post` state를 갱신한다.

공유 흐름:

1. 공유 버튼 클릭 시 `window.location.href`를 읽는다.
2. `navigator.clipboard.writeText(...)`로 복사한다.
3. 성공하면 `링크를 복사했습니다` 같은 짧은 상태 문구를 보여준다.
4. 실패하면 `주소를 복사하지 못했습니다`를 보여준다.

### 3.4 댓글 기능

새 프론트 API 파일 후보:

```text
frontend/src/api/comments.ts
```

필요 함수:

```ts
fetchComments(postId: string): Promise<Comment[]>
createComment(postId: string, body: string): Promise<Comment>
toggleCommentLike(commentId: string): Promise<{likeCount: number; myLike: boolean}>
```

새 타입 후보:

```ts
export interface Comment {
    id: string;
    body: string;
    authorId: number;
    createdAt: string;
    likeCount: number;
    myLike: boolean;
}
```

상세 페이지 댓글 흐름:

1. 게시글 상세 조회와 댓글 목록 조회를 함께 실행한다.
2. 댓글 작성 input 또는 textarea를 state로 관리한다.
3. 로그인 상태가 아니면 댓글 작성 영역에 로그인 안내를 보여준다.
4. 로그인 상태이면 댓글 작성 form을 보여준다.
5. 작성 성공 시 새 댓글을 댓글 목록에 추가한다.
6. 댓글 좋아요 클릭 시 해당 댓글의 `likeCount`, `myLike`만 갱신한다.

## 4. 백엔드 구현 계획

### 4.1 댓글 좋아요 조회

`GET /posts/{post_id}/comments`가 댓글마다 좋아요 상태를 내려주도록 확장한다.

비로그인:

- `myLike=false`

로그인:

- 현재 유저가 해당 댓글에 좋아요를 눌렀으면 `myLike=true`

좋아요 수:

- `votes.target_type = "comment"`
- `votes.target_id = comments.id`
- `votes.value = 1`
- 위 조건에 맞는 row 개수를 `likeCount`로 계산한다.

### 4.2 댓글 좋아요 토글

`POST /comments/{comment_id}/like`를 추가한다.

책임 분리:

- router: HTTP path, 로그인 의존성, 에러 변환
- service: 좋아요 추가/취소 규칙
- repository: comment 존재 확인, vote 조회, like count 계산

토글 규칙:

- 기존 vote가 없으면 `target_type="comment"`, `value=1`로 생성
- 기존 vote가 있으면 삭제
- 처리 후 현재 `likeCount`, `myLike` 반환

주의:

- 글 투표용 `POST /posts/{id}/vote`는 그대로 둔다.
- 댓글 좋아요는 싫어요 없이 좋아요만 지원한다.
- 새 테이블은 만들지 않는다.

## 5. 구현 순서

1. 타입 확장
   - `Post.score`, `Post.myVote` 추가
   - `Comment` 타입 추가
2. 프론트 API 추가
   - `votePost`
   - `fetchComments`
   - `createComment`
   - `toggleCommentLike`
3. 백엔드 댓글 좋아요 API 추가
   - 댓글 목록 응답 확장
   - 댓글 좋아요 토글 엔드포인트 추가
4. 메인 레이아웃 변경
   - 상단 검색/가입/로그인 헤더
   - 검색 실행
   - feed형 post card
5. 상세 레이아웃 변경
   - 게시글 투표 UI
   - 공유 버튼
   - 댓글 작성
   - 댓글 목록
   - 댓글 좋아요
6. 체크포인트 갱신
   - 이 계획 문서를 기준으로 구현이 끝난 범위만 기록한다.

## 6. 검증 계획

### 프론트

```bash
cd frontend
npm run build
npm run lint
```

확인 시나리오:

- 메인에서 검색어 입력 후 게시글 목록이 갱신된다.
- 비로그인 상태에서는 `가입`, `로그인` 버튼이 보인다.
- 로그인 상태에서는 `새 글`, `로그아웃` 버튼이 보인다.
- post card에서 상세 페이지로 이동할 수 있다.
- 상세에서 up/down 투표를 누르면 점수와 내 투표 상태가 바뀐다.
- 공유 버튼을 누르면 현재 URL이 복사된다.
- 비로그인 상태에서도 댓글 목록은 보인다.
- 로그인 상태에서 댓글을 작성할 수 있다.
- 댓글 좋아요를 누르면 좋아요 수와 내 좋아요 상태가 토글된다.

### 백엔드

예시 확인:

```bash
cd backend
source .venv/bin/activate
uvicorn app.main:app --reload
```

확인 시나리오:

- `GET /posts/{id}/comments`가 `likeCount`, `myLike`를 포함한다.
- 비로그인 댓글 조회에서는 `myLike=false`가 나온다.
- 로그인 후 `POST /comments/{comment_id}/like`를 호출하면 좋아요가 추가된다.
- 같은 요청을 한 번 더 보내면 좋아요가 취소된다.
- 기존 글 투표 API는 계속 정상 동작한다.

## 7. 학습 운영 메모

- 현재 프론트 8, 9단계 파일 일부는 코치 임시 채움 상태다.
- 이번 레이아웃 구현은 동작 확인을 위해 임시 구현 위에 확장할 수 있다.
- 이후 재드릴 때는 체크포인트의 `임시 채움 되돌리기` 목록을 기준으로 직접 다시 작성한다.
- 구현 중 새 파일을 만들면, 각 파일이 맡는 책임을 먼저 설명하고 진행한다.
