# ProjectLens 백엔드 전체 흐름 (라우터 → 서비스 → 레포지토리 → DB)

> 코어타임 발표용. 실제 코드와 1:1로 맞춘 문서다. 곁들인 다이어그램:
> - DB ERD: [ProjectLens_DB.drawio](ProjectLens_DB.drawio) (원본은 [backend/db/ALIBAI.drawio](../../backend/db/ALIBAI.drawio))
> - API 설계: [ProjectLens_API.drawio](ProjectLens_API.drawio)

## 1. 스택 한눈에

| 영역 | 기술 | 위치 |
|---|---|---|
| 웹 프레임워크 | FastAPI (async) | [app/main.py](../../backend/app/main.py) |
| DB 드라이버/ORM | SQLAlchemy 2.x async + `asyncpg` | [app/db.py](../../backend/app/db.py), [app/models.py](../../backend/app/models.py) |
| DB | PostgreSQL 16 + **pgvector** (docker `alibai-db`) | [backend/db/schema.sql](../../backend/db/schema.sql) |
| 인증 | bcrypt(passlib) + JWT(python-jose, HS256) | [app/auth/](../../backend/app/auth/) |
| AI | **OpenAI Agents SDK** + Responses API + Structured Outputs(Pydantic) | [app/ai/](../../backend/app/ai/) |
| RAG | OpenAI embeddings `text-embedding-3-small`(1536) + pgvector cosine(ivfflat) | [app/rag/](../../backend/app/rag/) |

## 2. 디렉터리·레이어 지도

```
backend/app/
├── main.py            # 앱 조립: 미들웨어(CORS/limiter) + 5개 라우터 include + /health
├── config.py          # .env 로드(Settings) — DB URL·JWT·OPENAI 키·RAG/MCP 파라미터
├── db.py              # async engine, SessionLocal, get_db() 의존성
├── models.py          # SQLAlchemy ORM 9개 테이블
├── schemas.py         # Pydantic 요청/응답 스키마 (camelCase alias)
├── limiter.py         # SlowAPI Limiter
├── routers/           # HTTP 경계: 검증 위임·예외→HTTP 변환 (auth/posts/comments/votes/analysis)
├── services/          # 도메인 로직: 검증·조합·트랜잭션 커밋 (posts/comments/votes/analysis_service)
├── repositories/      # DB 쿼리 캡슐화: 검색·태그·페이징·정렬은 전부 여기 (users/posts/comments/votes)
├── auth/              # security.py(해싱·JWT) + deps.py(토큰→User 가드)
├── ai/               # 히어로: agents/·runner.py·tools.py·prompts.py·context.py·schemas.py
├── rag/               # embedder.py·indexer.py·retriever.py·similarity.py
└── mcp_client/        # client.py(MCP 연결·로깅) + tools.py(allowlist 7종)
```

**레이어 규칙 (불변 계약):** 라우터는 얇게(인증·검증·예외 매핑만), 도메인 조립은 서비스,
**검색/태그/페이징/정렬·RAG 쿼리는 라우터가 아니라 repository 내부에** 둔다.

## 3. 요청 수명주기

모든 요청 공통: `Client → CORS+Limiter 미들웨어 → Router(Depends(get_db) 세션 주입) → Service → Repository → SQLAlchemy(asyncpg) → PostgreSQL`.
`get_db()`는 요청마다 세션을 열고 끝나면 반납한다:

```python
# app/db.py
async def get_db():
    async with SessionLocal() as session:
        yield session
```

### 3-1. 쓰기 경로 — `POST /posts`

1. **Router** [routers/posts.py:51](../../backend/app/routers/posts.py#L51): `get_current_user`로 인증 강제,
   `PostCreate`(body) 검증, 서비스 호출. `ValueError`는 400으로 변환.
   ```python
   post = await service.create(session, body, author_id=current_user.id)
   ```
2. **Service** [services/posts.py:8](../../backend/app/services/posts.py#L8): 태그 slug 검증(없는 태그면
   `ValueError`) → 레포로 INSERT → **RAG 인덱싱** → `commit()`.
   ```python
   post = await create_post(session, author_id=author_id, title=body.title, ...)
   await index_post_embedding(session, post)   # 임베딩 저장(같은 트랜잭션)
   await session.commit()
   ```
3. **Repository** [repositories/posts.py:102](../../backend/app/repositories/posts.py#L102): ORM `Post` 생성,
   `session.add` → `flush` → `refresh(["created_at"])` (커밋은 서비스가).
4. **RAG indexer** [rag/indexer.py:17](../../backend/app/rag/indexer.py#L17): 제목/한줄/본문/타깃/스택/태그를
   합쳐 임베딩 → `embeddings` 테이블에 upsert(기존 행 delete 후 insert).

→ 응답은 `PostOut`(camelCase). 새 글이라 `commentCount=0, myVote=0`.

### 3-2. 읽기 경로 — `GET /posts`

- **Router** [routers/posts.py:13](../../backend/app/routers/posts.py#L13): `get_current_user_optional`(비로그인 허용),
  쿼리 파라미터 `q·tag·postType·cursor`를 그대로 레포로.
- **Repository** [repositories/posts.py:37](../../backend/app/repositories/posts.py#L37):
  - 검색(`ilike` on title/body/one_liner), 태그 조인, 타입 필터, **커서 페이징**(`id < cursor`, `limit+1`로 다음 페이지 판단).
  - **N+1 방지**: 목록의 댓글 수와 "내 투표"를 각각 한 번의 `IN (...)` 집계 쿼리로 일괄 조회해 각 글에 붙인다.

## 4. 인증 흐름

- **회원가입** `POST /auth/signup` [routers/auth.py:12](../../backend/app/routers/auth.py#L12):
  username/email 중복검사 → `hash_password`(bcrypt) → 저장. 평문 비밀번호는 절대 저장하지 않는다.
- **로그인** `POST /auth/login` [routers/auth.py:26](../../backend/app/routers/auth.py#L26):
  OAuth2 form → `verify_password` → `create_access_token`(JWT, HS256, 만료 50분). **`@limiter.limit("5/minute")`** 로 무차별 대입 방어.
- **가드** [auth/deps.py](../../backend/app/auth/deps.py):
  - `get_current_user`: Bearer 토큰 디코드 → `sub`(username) → User 조회. 실패 시 401.
  - `get_current_user_optional`: 토큰 없거나 틀려도 `None`(조회 라우트용).
- **순수 함수** [auth/security.py](../../backend/app/auth/security.py): 해싱/검증/토큰 생성만 — DB·요청을 모른다.

## 5. 히어로 파이프라인 — AI 진단 리포트

진입점 `POST /posts/{id}/analysis`([routers/analysis.py:25](../../backend/app/routers/analysis.py#L25)).
동기 1종 + 비동기(`/jobs` 202 + `BackgroundTasks`) + `/status` + `/latest`.
**이 라우터는 별도 로그인 가드가 없다**(현재 구현 기준). 핵심 로직은
[services/analysis_service.py `run_analysis_for_post`](../../backend/app/services/analysis_service.py#L66):

1. **필수필드 게이트** `_missing_required_fields`: 제목 없거나, (외부 URL도 없고 설명도 200자 미만)이면
   모델을 부르지 않고 `need_more_info` 리포트로 단락.
2. **RAG** `index_post_embedding` + `retrieve_similar_projects`
   ([rag/retriever.py](../../backend/app/rag/retriever.py)): pgvector cosine로 유사 프로젝트 검색.
   인덱싱된 글이 충분(`rag_weighted_min_indexed_posts=20`)하면 의미유사도 0.65 + 태그/투표/최신/동일타입
   가중치로 재랭킹, 아니면 순수 cosine. 임계값 미만/표본 부족이면 빈 배열.
3. **Agent 실행** `run_project_analysis` ([ai/runner.py:82](../../backend/app/ai/runner.py#L82)):
   Agents SDK `Runner.run` → function tools가 MCP 근거를 수집 → **Responses API + Structured
   Outputs**(`ProjectAnalysisReport` Pydantic). 파싱 실패 시 1회 재시도, `ModelRefusalError`는 `refused`로.
   `OPENAI_API_KEY`가 없으면 mock 경로로 동작(로컬 검증용).
4. **MCP 근거 수집** ([ai/tools.py](../../backend/app/ai/tools.py) → [mcp_client/client.py](../../backend/app/mcp_client/client.py)):
   로컬 `MCPServerStdio`(`mcp-server/server.py`)를 띄워 도구 호출. 보안 가드는 6절 참조.
   도구 7종: `check_deploy_status·fetch_site_overview·fetch_site_context·fetch_rendered_site_overview·capture_screenshot·run_lighthouse_summary·fetch_github_readme`.
5. **폴백·실패강제**:
   - `_collect_rendered_site_fallback_if_needed`: HTTP 근거가 빈약하면 Chromium 렌더링 도구로 보강.
   - `_force_failed_if_external_evidence_unusable`: 사이트 접속 실패/차단인데 글 설명도 부족하면
     `completed`를 `failed`/`need_more_info`로 강등 → "안 되는 사이트에 허위 호평" 방지.
6. **저장** `_persist_analysis` ([analysis_service.py:243](../../backend/app/services/analysis_service.py#L243)):
   `ai_reports`(report JSON + model/usage/response_id/trace_id/input_snapshot/error) +
   `mcp_evidences`(도구별 호출 로그) + `index_ai_report_embedding`(리포트도 임베딩) 저장,
   `post.analysis_status`/`ai_summary` 갱신 후 commit.

흐름: `post → 게이트 → RAG + (Agent → MCP → Structured Output) → 폴백검증 → ai_reports/mcp_evidences/embeddings`.
