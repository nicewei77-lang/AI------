# 공식문서 레퍼런스 맵 (1차 출처)

> **목적:** AI에게 묻기 전에 **공식문서를 1차 출처로 직접 찾아보는 습관**을 들이는 연습. (지금은 연습 단계 — 개념→드릴 흐름은 유지하되, 아래 표시된 챕터는 "먼저 읽어보고 오기"를 시도한다.)
> **형태:** PDF로 받지 않고 **링크 + 버전 고정**으로 둔다(문서가 계속 갱신되므로). 특정 페이지만 추려 읽고, 통독하지 않는다.

---

## 사용법 — "공식문서 우선(doc-first)" 연습

1. 세션 시작 시 코치가 그 세션의 **📖 1차 출처 챕터**를 가리킨다.
2. 학습자가 그 챕터를 **먼저 직접 읽고**, "무엇이 뭔지/왜 그런지"를 자기 말로 한 번 요약한다.
3. 코치는 설명을 처음부터 다 깔지 않고 — **요약을 듣고 빈 곳·오해만 메운 뒤 드릴로 넘어간다.** (개념→드릴 골격은 그대로, 다만 개념의 "1차 소스"가 코치에서 문서로 이동.)
4. 막히면: 공식문서에서 **검색**으로 해당 키워드를 찾는 연습 → 그래도 막히면 코치 힌트.

> 처음부터 전 세션을 doc-first로 뒤집지 않는다. **B4(인증)** 를 첫 doc-first 트라이얼로 추천 — FastAPI의 OAuth2+JWT 챕터가 거의 그대로 베껴 쓸 만큼 자족적이라 "문서만 보고 따라 구현"의 성공 경험을 만들기 좋다.

---

## 1. MDN Web Docs — HTTP  *(프로토콜 기초, 프레임워크 중립)*

- 링크: https://developer.mozilla.org/en-US/docs/Web/HTTP  (에버그린)
- 역할: "이게 원래 어떻게 동작하나"의 바닥. 상태코드·CORS·인증·세션·쿠키의 *프로토콜 레벨* 정의. FastAPI 문서가 훨씬 빨리 읽히게 해주는 선행 지식.
- ALIBAI에 직접 쓰일 챕터:
  - **HTTP response status codes** (3xx/4xx/5xx) → B3
  - **CORS** → B4
  - **HTTP authentication / Cookies(세션 개념)** → B4 (Session vs JWT 대조용)
  - (개념만) HTTPS, WebSocket/SSE → 지금은 가볍게, 실시간은 범위 밖

## 2. FastAPI 공식 문서  *(백엔드 척추 — 단일 문서 커버리지 최고)*

- 링크: https://fastapi.tiangolo.com/  (최신 기준. 설치 버전과 맞춰 본다)
- 역할: REST 설계, 에러 핸들링, 의존성 주입(=레이어드 자연 강제), 설정, 인증까지 한 문서에. **세션마다 직접 펴서** 본다.
- ALIBAI에 직접 쓰일 챕터:
  - **Tutorial → First Steps / Path·Query Parameters** → B1, B3
  - **Settings and Environment Variables** (Pydantic Settings) → B1
  - **Request Body / Response Model** → B3
  - **Handling Errors** (`HTTPException`) → B3
  - **Dependencies** → B3~B4 (가드·DB 세션 주입)
  - **Security → "OAuth2 with Password (and hashing), Bearer with JWT tokens"** → B4 (ALIBAI 인증 거의 그대로)
  - **CORS (Middleware)** → B4
  - (stretch) **Testing** (`TestClient`) → 마무리

## 3. PostgreSQL 공식 문서  *(데이터 레이어 — 레퍼런스로 검색해 쓰기)*

- 링크: https://www.postgresql.org/docs/16/  (기획서 §4가 PG16 → **16으로 버전 고정**. `current`는 롤링이라 피함)
- 역할: SQL·JOIN·제약·인덱스·트랜잭션. 통독 X — Tutorial + SQL Language만 훑고 나머지는 검색.
- ALIBAI에 직접 쓰일 챕터:
  - **Part I. Tutorial** (1~3장: 기본, SQL, 고급) → B2 진입
  - **Part II. The SQL Language → Queries / Joins, Data Definition(제약·PK·FK·인덱스)** → B2
  - **Transactions** → B2~B3 (투표 토글 원자성)
  - (예고) pgvector는 이 위에 얹는 **확장(extension)** — AI 단계에서 이 문서 구조 위에 추가

## 4. SQLAlchemy 2.0 — ORM  *(파이썬 ↔ 테이블 매핑 — B2 모델·B3 쿼리)*

- 링크: https://docs.sqlalchemy.org/en/20/  (2.0 고정. 1.x 예제와 문법이 다르니 **반드시 `/en/20/`**)
- 역할: SQL 테이블을 파이썬 클래스로 선언(declarative)하고, 관계·제약·세션으로 다룬다. PostgreSQL이 "DB가 뭘 하나"라면, 여기는 "파이썬에서 그걸 어떻게 부르나".
- ALIBAI에 직접 쓰일 챕터:
  - **ORM → Declarative Mapping (`DeclarativeBase`, `Mapped`, `mapped_column`)** → B2 드릴 2 (지금)
  - **ORM → Relationship Configuration (`relationship`, `ForeignKey`, N:M `secondary`)** → B2~B3
  - **Describing Databases with MetaData → `UniqueConstraint`·복합 PK·`Index`** → B2
  - **ORM → Session / `AsyncSession` (asyncio extension)** → B3 (CRUD)
  - **Querying (`select()`)** → B3
