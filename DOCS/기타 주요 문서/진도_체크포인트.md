## 설계 결정 로그 (스펙과 달라진 계약 — 백엔드가 지금 씀)

- **투표(백엔드 필수):** 점수=net(up−down), 요청 `{value:1|-1}`, 응답 `{score,myVote}`. 같은 값 재클릭=취소, 반대 값=전환. `POST /posts/{id}/vote`, `votes` 테이블. 댓글은 평면.
- **JSON 계약(camel↔snake):** 외부 camelCase ↔ DB snake_case를 Pydantic alias로(`excuseText↔excuse_text`, `createdAt↔created_at`). 목록 태그 쿼리는 `?tag=`.
- **태그 ID:** 외부 `Tag.id`=문자열 slug(`t1`), DB FK=숫자 `tags.id`, `tags.slug UNIQUE`로 연결. 생성 스키마 내부 `tag_ids`/외부 alias `tagIds`/값 `string[]` slug.
- **Post.score** 백엔드 응답 포함. **임베딩 적재는 AI 단계로 분리**(B는 DB 저장만).
- (프론트 델타) `Post`: `situation` 제거·`title` 추가(분류는 tags 전담). `fetchPosts` 태그 파라미터명 `tagId`.
