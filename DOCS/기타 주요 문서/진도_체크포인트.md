## 설계 결정 로그 (스펙과 달라진 계약 — 백엔드가 지금 씀)

- **투표(백엔드 필수):** 점수=net(up−down), 요청 `{value:1|-1}`, 응답 `{score,myVote}`. 같은 값 재클릭=취소, 반대 값=전환. `POST /posts/{id}/vote`, `votes` 테이블. 댓글은 평면.
- **JSON 계약(camel↔snake):** 외부 camelCase ↔ DB snake_case를 Pydantic alias로(`serviceUrl↔service_url`, `oneLiner↔one_liner`, `createdAt↔created_at`). 본문은 `body`. 목록 태그 쿼리는 `?tag=`, 유형 필터는 `?postType=`.
- **태그 ID:** 외부 `Tag.id`=문자열 slug(`t1`), DB FK=숫자 `tags.id`, `tags.slug UNIQUE`로 연결. 생성 스키마 내부 `tag_ids`/외부 alias `tagIds`/값 `string[]` slug.
- **Post.score** 백엔드 응답 포함. **score=저장 컬럼 유지(A안 확정, 2026-06-13).** 투표 토글 트랜잭션 안에서 score를 ±갱신(votes 변경과 한 트랜잭션). JOIN은 태그필터·작성자에서 학습. 임베딩 테이블은 M0에 생성, 실제 적재/RAG 검색은 M4.
- (프론트 델타) `Post`: ProjectLens 필드(`body`, `serviceUrl`, `githubUrl`, `oneLiner`, `targetUser`, `techStack`, `analysisStatus`, `aiSummary`). `fetchPosts` 태그 파라미터명 `tagId`.
