## 설계 결정 로그 (스펙과 달라진 계약 — 백엔드가 지금 씀)

- **투표(백엔드 필수):** 점수=net(up−down), 요청 `{value:1|-1}`, 응답 `{score,myVote}`. 같은 값 재클릭=취소, 반대 값=전환. `POST /posts/{id}/vote`, `votes` 테이블. 댓글은 평면.
- **JSON 계약(camel↔snake):** 외부 camelCase ↔ DB snake_case를 Pydantic alias로(`serviceUrl↔service_url`, `oneLiner↔one_liner`, `createdAt↔created_at`). 본문은 `body`. 목록 태그 쿼리는 `?tag=`, 유형 필터는 `?postType=`.
- **태그 ID:** 외부 `Tag.id`=문자열 slug(`t1`), DB FK=숫자 `tags.id`, `tags.slug UNIQUE`로 연결. 생성 스키마 내부 `tag_ids`/외부 alias `tagIds`/값 `string[]` slug.
- **Post.score** 백엔드 응답 포함. **score=저장 컬럼 유지(A안 확정, 2026-06-13).** 투표 토글 트랜잭션 안에서 score를 ±갱신(votes 변경과 한 트랜잭션). JOIN은 태그필터·작성자에서 학습.
- **AI 분석 상태:** `ai_reports.status`는 `completed|need_more_info|failed|refused`, `posts.analysis_status`는 기존 CHECK 제약상 `refused` 미포함이라 `refused -> failed`로 매핑. 최신 리포트 없음은 `GET /posts/{id}/analysis/latest` 404.
- **M4/M5 RAG·도구:** 초기 유사도는 cosine 단독, 임계값 미만·데이터 부족은 빈 상태. fake/초기 시드 기준 `rag_similarity_threshold=0.18`; 데이터 쌓이면 Q4에서 재조정. `mcp_evidences.report_id`는 `ai_reports` 생성 후 붙이고, GitHub README는 owner/repo 파싱→`api.github.com` 고정 호출.
