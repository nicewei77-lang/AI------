## 설계 결정 로그 (스펙과 달라진 점 — 다음 세션이 헷갈리지 않게)

- **2026-06-07:** `Post`에서 `situation` 제거, `title` 추가. 변명 유형 분류는 `tags`가 전담.
  (이유: `situation`과 `tags`가 둘 다 "지각/결석/마감" 분류라 역할 중복. 게시판이니 제목이 필요.)
  → 코드(`types/post.ts`), 기획서 DB 스키마·기능표, 학습코치 AGENT 드릴 예시 모두 반영 완료.
- **2026-06-08:** `fetchPosts`의 태그 필터 파라미터 이름을 `tag` → `tagId`로 변경(`api/posts.ts`).
  (이유: `tag`가 Tag 객체처럼 읽혀 혼란. 실제 담는 값은 id 문자열이라 `t.id === params.tagId`로 의도가 명확해짐.)
  → 다음 주 axios 전환 시 URL 쿼리 키는 `?tag=`이지만 함수 본문에서 `{params:{tag: tagId}}`로 변환하면 되므로 바깥 시그니처 `tagId` 유지 가능. (프론트 지침서 예시와 기획서 JSON 계약에 반영 완료.)
- **2026-06-10 (백엔드 전환 + 피드백 반영):** ① Reddit형 **투표를 백엔드 필수**로 채택 → 기획서 §2.1·§6(`votes` 테이블)·§7(`POST /posts/{id}/vote`) 반영. 점수는 합산 net(up−down), 요청 body는 `{value:1|-1}`, 응답은 `{score,myVote}`. 같은 값 재클릭은 취소, 반대 값은 전환. 댓글은 평면(스레드·서브포럼 제외). ② **API JSON 계약**: 외부 camelCase ↔ DB snake_case를 Pydantic alias로 연결(`excuseText↔excuse_text`, `createdAt↔created_at`), 목록 태그 쿼리는 `?tag=`(프론트가 tagId→tag 변환). ③ **태그 ID 계약 확정**: 외부 `Tag.id`는 문자열 slug(`t1` 등), DB FK는 숫자 `tags.id`; DB `tags.slug UNIQUE`로 둘을 연결한다. 백엔드 생성 스키마는 내부 `tag_ids`, 외부 alias `tagIds`, 값은 `string[]` slug. 프론트 보충/axios 전환 때 `NewPost.tags`를 `tagIds`로 정리하거나 함수 본문에서 `tags.map(t=>t.id)`로 변환. ④ `Post.score`는 백엔드 응답에 포함하고, 프론트 투표 UI 보충 때 타입과 mock 기본값을 추가. ⑤ 코칭 계약 1차 출처를 `AGENTS.md`로, "정답은 요청 시 + 3단계 후 막히면 최소 공개" 문구를 AGENTS·백엔드 문서에서 일치시킴. ⑥ 기획서의 `POST /posts` 임베딩 적재는 AI 단계 작업으로 분리하고, B단계는 DB 저장만 한다.
